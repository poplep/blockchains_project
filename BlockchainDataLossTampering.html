Blockchains for Data Loss and Tampering Prevention


Blockchains are secure by nature as they use SHA-256 encryption for their data.

In order for an attacker to gain access to the data in one block, they need to decrypt the SHA-256 encryption for that block along with all of the other previous blocks since hashes are calculated using the current and previous hash of that block.

Security of a network and of data is never 100%. Therefore, the goal of a blockchain to prevent data loss and tampering is to greatly increase the amount of effort required to access the data in the hopes that it will deter attackers since they will deem the amount of work needed to access the data not worth the effort.

Blockchains are decentralized tamper-proof ledgers. Copies of the blockchain are stored on all network nodes instead of on a centralized location like a server. Therefore, if one node is compromised, then there are still various copies of the data on the network nodes.

Blockchains function on a consensus mechanism. Therefore, if an attacker wanted to change or add malicious data to the blockchain, then 51% of all nodes on the network would need to ensure that the data is valid before adding it to the blockchain. In this case, the data is malicious and therefore would not be added to the blockchain.

Blocks are constantly being mined on the blockchain whether or not data is actually being added. This is done to prevent against 51% attacks. 51% attacks occur when an attacker gains control of 51% of the nodes on the network and begin to maliciously alter or add blocks on the blockchain.

Blockchains, by nature, cannot be deleted or edited and are linked chronologically. Each block is timestamped and can be verified by the other nodes on the chain. Blockchains ensure data integrity by validating all copies of the blocks on the chain, ensuring full availability, providing tamper-proof security, and ensuring that all data can be linked to an irrefutable identity.
